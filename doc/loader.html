类加载

加载器类初始化时默认读取loader配置项，并加载其提供的加载规则。

除此之外，用户也可以通过Loader::add($type, $rules)方法添加规则

规则类型

prefix

prefix规则是以类名的namespace的第一级对应的目录的数组集合。类加载时只要找到其prefix对应的目录，除prefix外的namespace部分和基础类名部分则自动对应到子目录和文件名。

示例

如加载framework\core\http\Request类（framework是prefix，core\http是除prefix外的namespace部分，Request是基础类名），按上例规则则会加载FW_DIR/core/http/Request.php文件。

prefix与psr4规则类似，都是设置namespace与目录的对应关系，不同的是prefix只定义第一级namespace，而psr4不限定namespace层级，因此psr4更灵活，prefix更简洁，在合理规范的情况下prefix更合适，目前框架本身类加载只用到了prefix规则。

psr4

基于psr4规则的数组集合，具体规则参考这里。

alias

alias规则是以类别名对应类名的数组集合，通常用于简化类名。

示例

map

map规则是以类名对应文件名的数组集合，通常在少数类名不规律时使用。

file

直接加载的PHP文件名集合数组（非类加载），文件名忽略.php后缀。

Composer

框架支持composer并默认开启，可以方便的引用第三方扩展。不过框架核心并不依赖composer，所以在不需要使用时可以设置关闭。

composer的vendor目录通过环境常量app\env\VENDOR_DIR来设置，如果不设置此常量或设为null可以关闭composer使用。

composer加载规则的优先级小于框架内置的加载规则。
